# JourneyOS Future Space Refactor V1

This principle is frozen.

## Core Boundary

JourneyOS strictly separates:

- Reality
- Possibility

What has happened belongs to Memory.

What has not happened belongs to Future Space.

Future is never an extension of history. Future is possibility.

## Future Space

Future Space is not:

- a journey list
- a planner dashboard
- a calendar
- a project management surface

Future Space only answers:

```text
What may happen next?
```

## Future Object Lifecycle

```text
Idea
↓
Planning
↓
Confirmed
↓
Starting Soon
↓
Now
↓
Memory
```

Once a future object reaches Now, it leaves Future Space.

Once it has happened, it becomes Memory.

## Data Truth

Forbidden:

- future pretending to be memory
- AI-generated fake experiences
- automatically creating completed Moments
- showing completed demo/test journeys as future

Reality can only come from:

- user record
- photo
- location
- time
- voice
- direct writing

## Workspace

System-level controls belong to Workspace.

Workspace is **not** a main page, dashboard, or regularly accessed surface.

Workspace is the deepest layer for configuration and data preparation. It only surfaces when the user needs to:
- Configure system settings
- Prepare future journeys
- Manage data or APIs

### Workspace Scope

**Configuration:**
- Language settings
- Local backup/sync settings
- Data export/import
- Provider settings (API, Cloud)
- API keys management

**Data Import:**
- Future journey import (Excel, PDF, images, text)
- Travel data conversion
- Historical journey migration
- Travel planning files

### Workspace Principle

- **Hidden by default**: Only opened when needed
- **Minimal visual weight**: Never distract from memory
- **System thinking, not journey thinking**: Place all "technical" controls here
- **Never expose in daily experience**: Main app focuses on journeys, not configuration

### Access Pattern

Workspace is typically accessed via:
- Settings drawer
- Workspace section (collapsed by default)
- Import section (for bringing in future journeys)

Should not have its own main tab or prominent navigation button.

## Final Principle

Future keeps possibility.

Now keeps experience.

Memory keeps reality.

JourneyOS does not manufacture life.

JourneyOS preserves the life that really happened.
